# C Source Evaluation Index

Status:

- support-only
- no maintained runner
- not part of the default binary benchmark workflow

The source-code evaluation framework has been folded into the broader testing
documentation, but this directory is currently a dataset/template staging area
rather than an active execution path.

Canonical documentation:

- `Testing/TESTING_PLAN.md`

Source-eval support files remain here:

- `prompt_library/agent_task_prompts.md`
- `prompt_library/judge_prompts.md`
- `templates/task_spec.example.json`
- `templates/reference_annotations.example.json`
- `templates/judge_scorecard.example.json`
- `templates/run_manifest.example.json`

Use this directory for source-code benchmark datasets, prompt templates, and
machine-readable scorecards, but do not treat it as a parallel harness to the
binary benchmark. The authoritative automated path is still:

- `Testing/run_evaluation.py`
- `Testing/run_experiment_sweep.py`

Treat `Testing/TESTING_PLAN.md` as the authoritative evaluation spec.

## Dataset Generation

The datasets in this directory are for the **source-code evaluation track only**.

They are **not** required by:

- `Testing/run_evaluation.py`
- `Testing/run_experiment_sweep.py`

Those scripts operate on compiled binaries and prepared analysis bundles. The source-eval datasets are only needed when you want to benchmark the agent on C source code directly.

To generate or refresh the per-sample source datasets from the current prototype and experimental source corpora:

```bash
python Testing/C_Source_Evaluation/build_datasets.py --force
```

That script creates one dataset per real `.c` sample under `datasets/<program_id>/` and writes:

- `source/` copies of the input source file
- `reference/goal_spec.txt`
- `reference/summary_ref.md`
- `reference/annotations.json`
- `partial/main_snippet.c` for `T5` reconstruction tasks
- `tasks/*.json` task-spec scaffolds
- `datasets/manifest.json` as the generated dataset index

Important:

- The generated reference artifacts are seeded as `autogenerated_draft`.
- They are useful as starting points, but they should be reviewed before being treated as gold references for published evaluation results.
